\section{Resumen Ejecutivo}
% ════════════════════════════════════════════════════════════

El presente proyecto propone una arquitectura híbrida para estimar la concentración de NO$_2$, SO$_2$ y O$_3$ en puntos no muestreados de Santiago de Cali, combinando un modelo multimodal GeoVision-CLIP con interpolación geoestadística ST-Kriging. Se construyó un panel espacio-temporal de más de 50\,GB integrando datos de Sentinel-5P, Sentinel-2, ERA5-Land y las estaciones terrestres del DAGMA.

\subsection{Arquitectura del Modelo}

La función de pérdida total es:
\begin{equation}
  \mathcal{L}_{\text{total}} = \mathcal{L}_{\text{InfoNCE}} + \alpha\,(\mathcal{L}_{\text{sae\_img}} + \mathcal{L}_{\text{sae\_txt}})
\end{equation}
donde $\alpha=0.1$, temperatura aprendible $\tau=0.07$ e inicializada con pesos de RemoteCLIP (ViT-B/32).

\section{Discusión y Trabajo Futuro}
% ════════════════════════════════════════════════════════════

El modelo logra estimaciones continuas de contaminantes con incertidumbre cuantificada, cerrando el ciclo geoestadístico mediante un variograma de residuos nugget puro. Las principales limitaciones son: (i) cobertura temporal limitada de las estaciones DAGMA, (ii) ausencia de datos de PM$_{2.5}$ in-situ, y (iii) el downscaling de Sentinel-5P a resolución intraurbana introduce incertidumbre adicional.

\noindent\textbf{Trabajo futuro:} Incorporar datos AURA/OMI como fuente independiente de validación, implementar análisis de equidad espacial por estrato socioeconómico y extender el modelo a PM$_{2.5}$ usando MAIAC como proxy.

\section{Código Clave Comentado}
% ════════════════════════════════════════════════════════════

\begin{lstlisting}[caption={ST-Kriging: interpolación espacio-temporal con PyKrige.}]
from pykrige.ok3d import OrdinaryKriging3D
import numpy as np

def st_kriging(lats, lons, times, values, q_lats, q_lons, q_t,
               variogram='exponential'):
    lat_n = (lats - lats.mean()) / lats.std()
    lon_n = (lons - lons.mean()) / lons.std()
    t_n   = (times - times.mean()) / (times.std() + 1e-8)
    ok = OrdinaryKriging3D(lat_n, lon_n, t_n, values,
                           variogram_model=variogram, verbose=False)
    z, var = ok.execute('points',
                        (q_lats - lats.mean()) / lats.std(),
                        (q_lons - lons.mean()) / lons.std(),
                        (q_t - times.mean()) / (times.std() + 1e-8))
    return z, var  # valor predicho + varianza Kriging
\end{lstlisting}
